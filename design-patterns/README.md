# Design Patterns

Software design patterns commonly used in backend engineering and object oriented software development.

Design patterns help improve maintainability, extensibility and software design quality by providing reusable approaches to common engineering problems.

## Repository Structure

```text
design-patterns/

creational/
structural/
behavioral/
```

## Creational Patterns

Examples:

- Builder
- Singleton
- Factory Method
- Abstract Factory
- Prototype

---

## Structural Patterns

Examples:

- Decorator
- Adapter
- Proxy
- Composite
- Flyweight

---

## Behavioral Patterns

Patterns focused on communication and responsibility management between objects.

Examples:

- Observer
- Strategy
- Command
- State
- Iterator

---

## Why Design Patterns Matter

Design patterns commonly help improve:

- Maintainability
- Extensibility
- Code organization
- Reusability
- Software design quality

---

## Notes

Design patterns become increasingly valuable as backend systems evolve toward larger engineering environments and long term software ownership models.
